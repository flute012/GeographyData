地理空間資料整合平台｜產品邏輯與架構分析

使用者 (Users)
  • 地理科教師
  • 需要數據的研究者
核心問題 (Core Problems)
  • 資料分散、查找成本高。
  • 紙本單一、無法複製與取用。
  • 人工難以即時更新數據。
  • 人力查找容易出錯。
現況限制 (Constraints)
  • 無相關經驗與現成整合工具或技術。
  • 專案執行人力有限。
產品目標 (Product Goal)
讓教師能快速取得
可用數據
核心解法 (Core Solution)
  • API 串接：串接多個國際開放資料 API，集中抓取與更新。
  • 資料標準化：設計結構化資料模型（統一欄位、分類、單位），提升可查性。
  • 友善介面：做成可搜尋/可篩選的使用介面，支援教學、研究與命題。
產出價值 (Value Prop)
降低查找資料的時間
➔
提升教師出題的便利性與豐富性
關鍵功能 / 模組
資料搜尋 / 篩選 CSV 下載

  • 網站回訪率：例如每週使用次數。
  • 效率提升：內部人力時數釋放。
